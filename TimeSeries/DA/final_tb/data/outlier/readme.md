## readme
主动限幅算法主要有两个参数控制降噪力度：
1. outlier_window: 异常值处理窗口长度，表示在检测异常值时，考虑前后多少个数据点进行均值计算。
2. outlier_threshold: 异常值处理阈值，表示数据点偏离均值多少倍的标准差时被视为异常值。


在本文件夹中，针对数据集`data1220_1`算法降噪参数：
- 异常值处理窗口长度：`outlier_window` = 5
- 异常值处理阈值情况：
    | 阈值$n$ | 处理的异常数据点个数 | 总数据点个数 | 比例 | 
    | ------- | ------------------ | ------------ | ---- |
    | 1.8     | 0                  | 1160         | 0%   |
    | 1.75    | 143                | 1160         | ~12% |
    | 1.7     | 218                | 1160         | ~19% |
    | 1.6     | 383                | 1160         | ~33% |
    | 1.5     | 587                | 1160         | ~51% |
    | 1.4     | 970                | 1160         | ~84% |

本文件夹中，选取$n=1.4, n=1.75, n=1.8$三种阈值的降噪操作，分别得到一下降噪后的数据集：
- `data1220_1_n1.4.csv`: 使用阈值$n=1.4$进行降噪后的数据集
- `data1220_1_n1.75.csv`: 使用阈值$n=1.75$进行降噪后的数据集
- `data1220_1_n1.8.csv`: 使用阈值$n=1.8$进行降噪后的数据集
- `data1220_1.csv`: 原始数据集

三种数据集与原始数据集的对比图见：`Violet_outlier.svg`以及`Yellow_outlier.svg`
其中，$n=1.75$的数主动限幅方法，就是我们在报告和最终汇报（以及十五周汇报）中降噪配置 d1 的第一个组成部分。
